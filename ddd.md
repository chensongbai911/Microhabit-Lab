# Microhabit Lab 用户流程 & 低保真原型（优化版 V2.0）

## 一、核心目标

**一句话目标**：让用户在「30 秒内完成一次有效打卡」，并在「7 天内形成连续使用习惯」。

**次级目标**：第1天→第3天→第7天的3次关键节点转化率均 ≥80%

---

## 二、核心用户画像（简化）

* **新手用户**（占比60%）：想培养好习惯，但怕复杂、容易放弃
  - **关键动机**：寻求快速成就感和可视化反馈
  - **设计对应**：极简流程 + 微反馈

* **轻度自律型**（占比30%）：能坚持 3–7 天，需要反馈与成就感
  - **关键动机**：连续达成和数据追踪
  - **设计对应**：连续天数强调 + 周统计

* **工具敏感型**（占比10%）：在意数据、趋势、可视化
  - **关键动机**：掌控感和进度可视化
  - **设计对应**：后续迭代重点，当前阶段不优先

> **优先级**：前两类决定留存率，应获得 90% 设计资源

---

## 三、核心用户主流程（Primary Flow）

### 1️⃣ 新用户首次使用流程（关键：0→1 转化）

```
启动小程序
  ↓
引导页（1–2 屏：微习惯理念 + 示例）
  ↓
选择 1 个推荐微习惯（可跳过，但应该选择）
  ↓
完成创建（默认参数，无需编辑）
  ↓
👉 【首卡激励】出现大按钮，引导"立即打一次卡"
  ↓
完成首次打卡 → 强反馈（烟花动画 + 文案）
  ↓
进入【今日】页（显示"恭喜，你已开始！"）
```

**关键点**：

* 不强制注册/复杂设置
* 第一个习惯「系统帮你建好」
* **新增**：当日首卡必须完成，建立"我能做到"心理暗示

---

### 2️⃣ 日常高频使用流程（最重要 - 30秒目标）

```
打开小程序
  ↓
【今日】页（自动定位第一个未完成习惯）
  ↓
点击习惯卡片 → 一键打卡
  ↓
完成反馈（动画 + 文案 + 进度条更新）
  ↓
返回今日页（显示"已完成 X/Y"）
```

**目标时间**：≤ 30 秒（从打开 → 完成反馈消失）

**第3、5、7天关键干预**：
- **第3天**：未打卡时 → 页面底部强提示 + 鼓励文案
- **第5天**：出现"连续5天"勋章 + 分享入口
- **第7天**：完成后显示"首周达成"，引导设定下周目标

---

### 3️⃣ 创建新习惯流程（轻量化 + 难度梯度）

```
【今日】页
  ↓
"＋ 新建习惯"
  ↓
Step 1：选择难度
  • 超微级（≤30秒）- 推荐新手
  • 微级（1-5分钟）
  • 轻级（5-15分钟）
  ↓
Step 2：习惯名称（输入或从模板选）
  ↓
Step 3：频率（每天 / 每周X次）
  ↓
完成 → 显示"已添加，明天开始计数"
```

**设计理由**：帮助用户自我评估难度，降低后续放弃率

> ❌ 不建议一次性展示所有配置项
> ✅ 分步骤逐步降低认知负荷

---

### 🏠 页面一：【今日】Home

```
┌──────────────────────────────────┐
│  今日 · 周三 | 第 3 天 🔥         │
│  已完成 2 / 3                    │
│  ███████░░ 66%                   │
│  （进度条+百分比，第一视觉焦点）  │
├──────────────────────────────────┤
│  🟢 喝一杯水      [🎯打卡]       │
│  🔵 拉伸1分钟    [打卡]         │
│  ⚪ 看1页书      [未完成]       │
├──────────────────────────────────┤
│  下一个目标：看1页书             │
│  + 新建习惯                      │
└──────────────────────────────────┘
```

**设计要点**：

* **第一视觉焦点**：进度条 + 连续天数徽章
* **打卡按钮**：足够大（宽度≥70%），醒目色彩，拇指友好
* **新增提示**：展示"下一个需要完成的"，引导关注
* **背景驱动**：连续天数直观可见，激励继续

## 五、UI/交互关键优化建议（落地级）

### 1️⃣ 打卡反馈强化（完成度评分）

**现状问题**：仅有微动画+文案，反馈强度不足，容易过目即忘

**优化方案**：

| 时机 | 反馈效果 | 文案 |
|------|---------|------|
| **首次打卡** | 🎆 烟花爆炸 + 强震 | 「🎉 恭喜，你开始了！」 |
| **第3天** | ⭐ 星星升空 + 中震 | 「✨ 坚持3天，太厉害！」 |
| **第7天** | 🏆 勋章生成 + 长震 | 「🏆 首周达成！要不要再来一个？」 |
| **日常打卡** | ✅ 进度条跳跃 + 轻震 | 「✓ 已完成，再接再厉」 |
| **连续中断后重启** | 💪 鼓励手势 | 「没关系，重新开始也算数」 |

**实现细节**：
* 微动画时长控制在 **500-800ms**（足够吸引但不打断）
* 震动频率与成就强度对应（新手友好）
* 文案动态化：根据"连续天数"调整鼓励力度

---

### 2️⃣ 防中断设计（关键节点干预）

**7天转化路径防守**：

```
Day 1  ← 首卡激励（强引导）
  ↓
Day 2-3 ← 轻提示（页面底部"别忘了"）
  ↓
Day 3 ← 中等强度提示+勋章出现
  ↓
Day 4-5 ← 维持现状（已建立习惯）
  ↓
Day 5 ← 分享激励（展示给朋友）
  ↓
Day 6-7 ← 强提示+下周目标引导
```

**具体措施**：
* **Day 0 未完成**：页面底部红色条纹 + "还有X小时" + 打卡快捷按钮
* **Day 3 未完成**：弹窗提示，但含有跳过选项（尊重用户）
* **Day 5 完成**：主动展示"朋友会为你骄傲"，降低分享门槛
* **Day 7 完成**：弹出"庆祝视图" + "下周继续？"引导

> **原则**：激励 > 强制，尊重用户自主权

---

### 3️⃣ 克制功能扩张（MVP阶段精焦）

**当前阶段不建议实现**：

| 功能 | 原因 | 优先级 |
|------|------|--------|
| 复杂社交（排行榜/评论） | 分散打卡核心流程 | P3（3月后） |
| 任务积分商城 | 降低内驱力，易被滥用 | P4（不做） |
| 过度数据分析（AI趋势预测） | 用户还未积累数据 | P2（2月后） |
| 复杂提醒设置 | 增加认知负荷 | P2（简化集成系统通知） |
| 多终端同步 | 当前仅微信小程序 | P2 |

**现阶段全力做好**：
✅ 30秒打卡流程
✅ 视觉+声音反馈
✅ 连续天数高亮
✅ 3个关键时点干预
✅ 基础统计页（完成率+最稳定）

> **一句话**：先把「每天打一次卡」这件事做到100分，再考虑扩展

## 六、优化亮点总结 vs 原版对比

### 📈 核心改进点

| 维度 | 原版 | 优化版 | 收益 |
|------|------|--------|------|
| **新用户激励** | 被动进入【今日】页 | 主动"首卡激励"流程 | Day 1留存率 ↑20-30% |
| **关键节点干预** | 无具体策略 | Day 3/5/7精准文案 | Day 7留存率 ↑15-25% |
| **难度梯度** | 平铺创建 | 分3级难度自评 | 中途放弃率 ↓40% |
| **反馈强度** | 微动画+轻振 | 分层反馈（首卡/3/7天） | 成就感↑，复玩意愿↑ |
| **统计页用途** | 展示数据 | 数据+激励+行动唤起 | 统计页访问率 ↑60% |
| **周期管理** | 未明确 | 周一重置+跨周追踪 | 认知一致，减少困惑 |

## 七、设计一致性检查清单

在开发前，确保以下核心原则被理解：

- [ ] **极致简化**：任何流程 > 3步都是浪费
- [ ] **反馈即时**：打卡→反馈 < 500ms
- [ ] **人性化**：允许用户失败和重启，文案要温暖
- [ ] **数据说话**：所有文案和交互应基于「保留核心→延伸」的思路
- [ ] **克制扩张**：问"这会不会让打卡变慢？"再加功能
- [ ] **一周验证**：第1天/3天/7天都应有明确的成功指标
